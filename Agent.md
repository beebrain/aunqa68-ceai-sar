# 🤖 Agent Rules & Permissions (Agent.md)

เอกสารนี้กำหนดกฎระเบียบ ขอบเขตการทำงาน และการอนุญาตสิทธิ์ (Authorization) สำหรับ AI Coding Agent (Antigravity/Claude) ในการเข้าถึงและแก้ไขข้อมูลในพื้นที่โปรเจกต์ **AUN-QA SAR (Computer Engineering & AI)**

---

## 1. ขอบเขตสิทธิ์การใช้งานของ Agent (Permitted Scopes)

Agent ได้รับอนุญาตให้ดำเนินการภายใต้ขอบเขตดังต่อไปนี้:

### 🔑 คำสั่งอนุญาตพิเศษ (Special Override Command)
*   **คำสั่ง `allow_all`:** หากผู้ใช้พิมพ์คำสั่งหรือข้อความ `allow_all` หมายถึงการ **อนุญาตทุกการทำงานแบบครอบคลุม** ซึ่งรวมถึงการ **อนุญาตให้รัน Python และสคริปต์ทุกประเภท (Allow running python and all types of scripts)**, การแก้ไขไฟล์, การเข้าถึงข้อมูล, และการรันคำสั่ง Terminal ทั้งหมด โดยที่ Agent **ไม่ต้องถามยืนยันสิทธิ์หรือขอความเห็นชอบใดๆ อีก** และสามารถดำเนินกระบวนการทำงานแบบอัตโนมัติได้อย่างรวดเร็ว


### 📂 การจัดการไฟล์และโฟลเดอร์ (File Operations)
*   **อนุญาตให้เขียนและแก้ไข (Write & Modify):**
    *   `/aun-qa-wiki/` (สำหรับการอัปเดตข้อมูล Wiki, Guidelines, Summaries, Raw extracts และ Index)
    *   `/sections/` (ไฟล์ LaTeX `.tex` สำหรับเนื้อหาแต่ละ Criterion)
    *   `/มคอ3/` และ `/มคอ5/` (เอกสารหลักสูตร)
    *   `SAR-CEAI-2568.tex` และไฟล์กำหนดค่าอื่นๆ ใน Root directory
*   **ไม่อนุญาตให้ออกนอกพื้นที่ทำงาน:** ห้ามเข้าถึง แก้ไข หรือเขียนไฟล์ภายนอก Workspace หรือโฟลเดอร์ระบบ (`/tmp`, `/etc`, ฯลฯ) เว้นแต่ได้รับคำสั่งเฉพาะเจาะจงจากผู้ใช้

### 🌐 การเข้าถึงเครือข่ายภายนอก (Network Access)
*   **เว็บไซต์หลักที่อนุญาต (Whitelisted Domains):**
    *   `https://aunqa.uru.ac.th/` (ระบบฐานข้อมูล AUN-QA มหาวิทยาลัยราชภัฏอุตรดิตถ์)
    *   `https://syweb.kidcbc.work/` (ระบบจัดการภายนอกที่เกี่ยวข้อง)
*   **พฤติกรรมการเรียกใช้:**
    *   อนุญาตให้ดึงข้อมูลหน้าเว็บ (HTTP GET) และส่งคำขอค้นหา (HTTP POST) เพื่อสะกัดเอกสารหลักฐานและข้อมูลประกอบการประเมิน
    *   หลีกเลี่ยงการโจมตีหรือการเรียกข้อมูลที่ถี่เกินไปจนเป็นภาระต่อเซิร์ฟเวอร์ปลายทาง (Rate limiting)

### 💻 การรันคำสั่ง Terminal (Command Execution)
*   **คำสั่งที่ได้รับสิทธิ์รันโดยตรง (Pre-approved or Normal Usage):**
    *   `python3` (สำหรับสคริปต์วิเคราะห์ข้อมูล, ดึงข้อมูลพอร์ทัล และ Ingest ข้อมูล)
    *   `git` (สำหรับการตรวจสอบประวัติ บันทึกการเปลี่ยนแปลง `git add`, `git commit`)
    *   `latexmk` / `pdflatex` (สำหรับการตรวจสอบและ Compile รายงาน SAR เป็น PDF)
    *   `grep` / `find` (สำหรับค้นหาข้อความและหลักฐานในระบบ)

## 2. แนวทางการทำงานร่วมกันระหว่างผู้ใช้และ Agent (Co-working Guidelines)

1.  **การเขียนโค้ดและสคริปต์:**
    *   หลีกเลี่ยงการใช้ไลบรารีภายนอกที่ต้องติดตั้งเพิ่มเติม (เช่น `requests`, `beautifulsoup4`) หากไม่จำเป็น
    *   **แนะนำ:** ให้ใช้ไลบรารีมาตรฐานของ Python (เช่น `urllib.request`, `html.parser`, `json`, `re`) เพื่อความสะดวกและไม่ต้องการสิทธิ์ผู้ดูแลระบบในการติดตั้งแพ็กเกจ
2.  **การปฏิบัติตามมาตรฐาน AUN-QA v4.0:**
    *   ความสอดคล้องเชิงโครงสร้าง (Constructive Alignment) ระหว่าง PLO, YLO, และ CLO เป็นสิ่งที่สำคัญที่สุด
    *   เมื่อดึงข้อมูลหลักฐานจากระบบพอร์ทัล ต้องนำมาจัดระเบียบให้เข้ากับเกณฑ์ย่อยของ Criterion 7 (Facilities) และ Criterion 8 (Outcomes) อย่างเป็นระบบ
3.  **ความโปร่งใส (Transparency):**
    *   ห้ามสร้างลิงก์หรือหลักฐานขึ้นมาเอง (Fake Evidence)
    *   หลักฐานทุกชิ้นใน Wiki ต้องอ้างอิงแหล่งที่มา เช่น URL จาก `aunqa.uru.ac.th` หรือเอกสารในพื้นที่หลักฐานจริงเท่านั้น

---

## 3. การจัดการความปลอดภัยของข้อมูล (Security Safeguards)

*   **CSRF Tokens & Session:** เมื่อสคริปต์ของ Agent จำเป็นต้องดึงข้อมูลผ่าน POST request ต้องมีการสกัด CSRF Token อย่างถูกต้อง และไม่มีการบันทึกรหัสผ่านของผู้ใช้ลงในโค้ด
*   **Git Commits:** แนะนำให้ทำ Git commit ทุกครั้งหลังเสร็จสิ้นกระบวนการ Ingest ข้อมูลชุดใหญ่ เพื่อป้องกันข้อมูลสูญหายและให้ผู้ใช้ตรวจสอบย้อนหลังได้สะดวก
