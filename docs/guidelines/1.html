Criterion 1 — Expected Learning Outcomes (PLOs/CLOs)

Criterion 1 — Expected Learning Outcomes (PLOs/CLOs)

ที่มา: เอกสารอบรม ผศ.ดร.ขวัญดาว (25 มี.ค. 2569)

Requirements Overview

ข้อ ข้อกำหนด (สรุป)
1.1 PLOs ถูกสร้างตาม learning taxonomy, สอดคล้อง vision/mission, สื่อสารกับ SHs ทุกกลุ่ม
1.2 CLOs ของทุกรายวิชาเหมาะสม และสอดคล้องกับ PLOs
1.3 PLOs ประกอบด้วยทั้ง Generic LOs และ Subject-Specific LOs
1.4 PLOs สร้างจาก Needs/Requirements ของ SHs โดยเฉพาะ SHs ภายนอก
1.5 หลักสูตรแสดงให้เห็นว่าผู้เรียนบรรลุ PLOs เมื่อสำเร็จการศึกษา

R 1.1 — PLOs ตาม Learning Taxonomy + สอดคล้อง Vision/Mission + สื่อสาร SHs

เนื้อเกณฑ์ที่พึงพิจารณา:

  1. Learning Taxonomy — PLOs ต้องขึ้นต้นด้วย Action Verb, มี Learning Level ชัดเจน, วัดและประเมินได้ เป็นรูปธรรม
  2. Vision/Mission — PLOs สอดคล้องกับวิสัยทัศน์ พันธกิจ และอัตลักษณ์ของมหาวิทยาลัย/คณะ/หลักสูตร
  3. สื่อสาร SHs — หลักสูตรกำหนด SHs ที่ต้องสื่อสาร PLOs โดยจะสื่อสาร หลังผ่านสภาอนุมัติหลักสูตรแล้ว

หลักฐาน/เอกสารที่ควรมี:
- เอกสาร PLOs พร้อม Action Verb และ Learning Level
- เอกสารแสดงความสัมพันธ์ PLOs ↔ Vision/Mission
- แผนการสื่อสาร PLOs ไปยัง SHs แต่ละกลุ่ม

R 1.2 — CLOs ของทุกรายวิชาสอดคล้องกับ PLOs

เนื้อเกณฑ์ที่พึงพิจารณา:

  1. CLOs ต้องสร้างอย่างเหมาะสม: ชัด เป็นรูปธรรม วัดได้ และสอดคล้องกับ PLOs (นำ PLOs มาเป็นตัวตั้งต้น แตกออกเป็น CLOs)
  2. ต้องสอดคล้องทั้ง เชิงสาระ และ ระดับการเรียนรู้ (Learning Level ของ CLOs ต้องสอดคล้องกับ Learning Level ของ PLOs)
  3. ทุกรายวิชา ต้องมี CLOs (อย่างน้อยทุกรายวิชาที่เปิดสอน)

หลักฐาน/เอกสารที่ควรมี:
- ตารางการกระจาย PLOs → CLOs ของทุกรายวิชา (Curriculum Mapping)
- มคอ.3 ของทุกรายวิชา พร้อม CLOs ชัดเจน

R 1.3 — PLOs มีทั้ง Generic LOs และ Subject-Specific LOs

เนื้อเกณฑ์ที่พึงพิจารณา:

  • PLOs ต้องมีทั้ง SSLOs (Subject Specific Learning Outcomes) และ GLOs (Generic Learning Outcomes)
  • Generic LOs: written/oral communication, problem-solving, IT, teambuilding skills ฯลฯ

หลักฐาน/เอกสารที่ควรมี:
- ตารางแสดง Generic LOs vs Subject-Specific LOs แยกแต่ละ PLO

R 1.4 — PLOs สร้างจาก Needs ของ SHs โดยเฉพาะ SHs ภายนอก

เนื้อเกณฑ์ที่พึงพิจารณา:

  1. กำหนด SHs ที่ต้องการเก็บข้อมูล (ต้องมี SHs นอกมหาวิทยาลัย)
  2. กำหนดวิธีเก็บข้อมูล จาก SHs แต่ละกลุ่ม
  3. วิเคราะห์/สังเคราะห์ → Sum Needs → นำมาจัดทำ PLOs
  4. แสดง mapping ว่า PLO ข้อไหนสะท้อน requirement/need ใด
  5. SHs ใน R1.4 ควรเป็นกลุ่มเดียวกับ R1.1

ข้อสังเกต: ผู้ให้ข้อมูลอาจเคยหรือไม่เคยใช้บัณฑิตของเรา แต่เราอยากให้บัณฑิตไปทำงานในหน่วยงานนั้น

หลักฐาน/เอกสารที่ควรมี:
- รายงานผลการสำรวจ Needs จาก SHs
- ตารางแสดงความสัมพันธ์ "ความต้องการ SHs ↔ PLOs"

R 1.5 — ผู้เรียนบรรลุ PLOs เมื่อสำเร็จการศึกษา

เนื้อเกณฑ์ที่พึงพิจารณา:

  1. กำหนด เวลาที่จะวัด การบรรลุ PLOs แต่ละข้อ (ควรวัดตอนที่นศ.อยู่กับเรา ไม่ปะปนกับสิ่งที่ได้รับจากการทำงาน เช่น ถาม นศ.ปีสุดท้าย, บัณฑิตที่เพิ่งจบ, นายจ้างของศิษย์เพิ่งจบ)
  2. กำหนด วิธีการ / เครื่องมือ / เกณฑ์การตัดสิน ว่าบรรลุหรือไม่

กรอบ 4 มิติการวัด:
- When to assess: ช่วงเวลาที่จะบรรลุแต่ละ PLO (ปี 3, ปี 4, หรือเมื่อสำเร็จการศึกษา)
- Method: How to assess (Exit Exam, Self-assessment, ใช้ผลการสอบรายวิชา)
- Tool: What is used to assess
- Person: Who assesses

ตัวอย่างวิธีวัด: Exit Exam, นศ.ประเมินตนเอง, ใช้รายวิชาเป็นตัวกำหนด (เช่น วิธี 3+1 หรือ 3+2)

หลักฐาน/เอกสารที่ควรมี:
- ตารางแสดง PLOs + วิธีการวัด + เครื่องมือ + เกณฑ์ตัดสิน + ชั้นปีที่บรรลุ

Addendum (สำหรับหลักสูตรบัณฑิตศึกษา)

  • R1.5: "achieved by the time they graduate" อาจรวมถึง contribution หลังจบการศึกษาในระดับปริญญาเอก
  • R1.1/1.3: PLOs ต้องพิจารณา national/regional/global reference points ของ postgraduate programme รวมถึง original research, novelty, emerging theories
