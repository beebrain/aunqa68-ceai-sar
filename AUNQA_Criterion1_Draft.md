## ภาพรวม (Overview)

หลักสูตรวิศวกรรมศาสตรมหาบัณฑิต สาขาวิชาวิศวกรรมคอมพิวเตอร์และปัญญาประดิษฐ์ มหาวิทยาลัยราชภัฏอุตรดิตถ์ (พ.ศ. 2568) ได้กำหนดและพัฒนาผลการเรียนรู้ระดับหลักสูตร (Programme Learning Outcomes: PLOs) ที่ครอบคลุมทั้ง Generic Learning Outcomes และ Subject-Specific Learning Outcomes โดยอ้างอิง Bloom's Revised Taxonomy สอดคล้องกับวิสัยทัศน์และพันธกิจของมหาวิทยาลัย และสะท้อนความต้องการของผู้มีส่วนได้ส่วนเสียทั้งภายในและภายนอก

## Sub-criterion 1.1 — PLOs ตาม Learning Taxonomy + Vision/Mission + สื่อสาร Stakeholders

### คำอธิบาย (Description)

หลักสูตรได้กำหนด PLOs โดยใช้หลัก **Bloom's Revised Taxonomy** ควบคู่กับรูปแบบการเขียนวัตถุประสงค์แบบ ABCD (Action verb, Behavior, Condition, Degree) เพื่อให้ผลการเรียนรู้มีความชัดเจน วัดผลได้ และสะท้อนระดับการเรียนรู้ที่คาดหวัง PLOs ทุกข้อขึ้นต้นด้วย Action Verb ที่ระบุระดับ Bloom ไว้อย่างชัดเจน ดังนี้

| PLO | ข้อความผลการเรียนรู้ (ย่อ) | Bloom's Level | Bloom Code |
|-----|--------------------------|---------------|------------|
| PLO1 | ใช้ซอฟต์แวร์/ฮาร์ดแวร์สร้างสรรค์ผลงานและนวัตกรรม | Creating | C6 |
| PLO2 | ประยุกต์ความรู้แก้ปัญหาขององค์กร ชุมชน และท้องถิ่น | Applying | C3 |
| PLO3 | สร้างองค์ความรู้/ต้นแบบนวัตกรรมด้าน CE&AI | Creating | C6 |
| PLO4 | ปฏิบัติตามหลักจริยาบรรณวิชาชีพวิศวกรรมคอมพิวเตอร์ | Evaluating | C5 |
| PLO5 | คิดวิเคราะห์ สื่อสาร ทำงานร่วมกัน และเรียนรู้ตลอดชีวิต | Analyzing | C4 |

PLOs ดังกล่าวได้รับการเชื่อมโยงกับ **วิสัยทัศน์ พันธกิจ และปรัชญา IEL** ของมหาวิทยาลัยราชภัฏอุตรดิตถ์ ดังตารางต่อไปนี้

| ข้อความทิศทาง (สรุป) | PLO1 | PLO2 | PLO3 | PLO4 | PLO5 |
|----------------------|:----:|:----:|:----:|:----:|:----:|
| วิสัยทัศน์ มร.อ.: คุณภาพ พันธกิจสัมพันธ์ พัฒนาท้องถิ่น | ✅ | ✅ | ✅ | ✅ | ✅ |
| พันธกิจ: กำลังคนดิจิทัล–AI และแก้ปัญหาในพื้นที่ | ✅ | ✅ | ✅ |    | ✅ |
| ปรัชญา IEL: ประสบการณ์เชิงบูรณาการ | ✅ | ✅ | ✅ | ✅ | ✅ |

กระบวนการพัฒนา PLO ผ่านการกลั่นกรองตามกลไกกำกับคุณภาพ ได้แก่ คณะกรรมการพัฒนาหลักสูตร → คณะกรรมการบัณฑิตศึกษาประจำมหาวิทยาลัย → สภาวิชาการ → **สภามหาวิทยาลัย (มติครั้งที่ 4/2568 วันที่ 4 เมษายน พ.ศ. 2568)**

หลักสูตรได้เผยแพร่ PLOs ผ่านช่องทาง ได้แก่ **เว็บไซต์หลักสูตร** ([syweb.kidcbc.work](https://syweb.kidcbc.work)) เว็บไซต์มหาวิทยาลัยราชภัฏอุตรดิตถ์ เว็บไซต์คณะเทคโนโลยีอุตสาหกรรม **เพจ Facebook หลักสูตร CE&AI** ([facebook.com/profile.php?id=61577236346855](https://www.facebook.com/profile.php?id=61577236346855)) และเอกสารแนะนำหลักสูตรสำหรับผู้สมัคร

### จุดแข็ง (Strengths)
- PLOs ออกแบบตาม Bloom's Revised Taxonomy ระดับสูง (C3–C6) เหมาะสมกับบัณฑิตศึกษา
- ผ่านกลไกกำกับคุณภาพหลายระดับก่อนอนุมัติ ตรวจสอบย้อนกลับได้จากมติสภา

### จุดที่ต้องพัฒนา (Areas for Improvement)
- ยังขาดหลักฐานเชิงประจักษ์การเผยแพร่ PLO (screenshot เว็บไซต์จริง, คู่มือนักศึกษาฉบับพิมพ์)
- รายงานการประชุม กมศ. (AUNQA-1-2) ยังไม่มีรายละเอียดกระบวนการ Bloom/ABCD analysis

### หลักฐานประกอบ
- **AUNQA-1-1** มคอ.2 ฉบับผ่านสภา ✅
- **AUNQA-1-2** รายงานการประชุมคณะกรรมการประจำหลักสูตร ⚠️
- **AUNQA-1-4** มติสภามหาวิทยาลัย ครั้งที่ 4/2568 ✅

## Sub-criterion 1.2 — CLOs สอดคล้องกับ PLOs (Constructive Alignment)

### คำอธิบาย (Description)

PLOs ของหลักสูตรได้ถูกถ่ายทอดกระจายลงสู่ **Course Learning Outcomes (CLOs)** ของทุกรายวิชาบังคับ 6 รายวิชา และวิทยานิพนธ์/สารนิพนธ์ ผ่านกระบวนการ Constructive Alignment โดยอาจารย์ผู้รับผิดชอบรายวิชาแต่ละท่านกำหนด CLO ตามหลัก ABCD และเชื่อมโยงเข้ากับ PLO อย่างน้อย 1 ข้อ

**ตาราง CLO–PLO Mapping (วิชาบังคับ)**

|---------|:----:|:----:|:----:|:----:|:----:|

| 7015906 ระเบียบวิธีวิจัยฯ |    | ✅ | ✅ | ✅ | ✅ |
| 7015907 สัมมนาวิศวกรรม CE-AI |    | ✅ | ✅ | ✅ | ✅ |
| 7015101 เทคโนโลยีอุบัติใหม่ฯ | ✅ | ✅ |    |    |    | ← รหัสยืนยันจาก wiki + มคอ.3|
| 7015908 หัวข้อพิเศษฯ | ✅ | ✅ |    | ✅ |    |

PLO ทุกข้อได้รับการรองรับจาก CLO ของรายวิชาในหลักสูตร ระดับ Bloom ของ CLO สอดคล้องกับระดับของ PLO ที่รองรับ (เชิงสาระและเชิงระดับความซับซ้อน) ตามหลักการ Constructive Alignment ของ Biggs (2003)

### จุดแข็ง (Strengths)
- PLO ทุกข้อ (PLO1–PLO5) มี CLO รองรับครบทุกข้อในวิชาบังคับและวิทยานิพนธ์/สารนิพนธ์
- CLO เขียนด้วย Action Verb ที่วัดได้ ระบุระดับ Bloom ชัดเจน

### จุดที่ต้องพัฒนา (Areas for Improvement)
- CLO ของวิชาเลือก 21 วิชายังอยู่ในรูปตาราง matrix เท่านั้น ยังไม่มี full CLO text สำหรับทุกรายวิชา
- ควรจัดทำตาราง Curriculum Mapping แบบสมบูรณ์ครอบคลุม 26 วิชา (บังคับ + เลือก)

### หลักฐานประกอบ
- **AUNQA-1-3** รวมเล่ม CLO รายวิชา พร้อม CLO–PLO Mapping ✅

## Sub-criterion 1.3 — Generic Learning Outcomes และ Subject-Specific Learning Outcomes

### คำอธิบาย (Description)

PLOs ของหลักสูตรประกอบด้วยทั้ง Generic Learning Outcomes (GLOs) และ Subject-Specific Learning Outcomes (SSLOs) ครบถ้วน

| PLO | คำอธิบายผลการเรียนรู้ | ประเภท | Bloom Level |
|-----|----------------------|--------|-------------|
| PLO1 | ใช้ซอฟต์แวร์/ฮาร์ดแวร์สร้างสรรค์ผลงาน นวัตกรรม | Subject-Specific | C6 |
| PLO2 | ประยุกต์ความรู้แก้ปัญหาขององค์กร/ชุมชน/ท้องถิ่น | Subject-Specific | C3 |
| PLO3 | สร้างองค์ความรู้/ต้นแบบนวัตกรรม CE&AI | Subject-Specific | C6 |
| PLO4 | ปฏิบัติตามจริยาบรรณวิชาชีพวิศวกรรมคอมพิวเตอร์ | **Generic** | C5 |
| PLO5 | คิดวิเคราะห์ สื่อสาร ทำงานร่วมกัน เรียนรู้ตลอดชีวิต | **Generic** | C4 |

**Generic Learning Outcomes** ครอบคลุมตามกรอบที่ AUN-QA v4.0 กำหนด:
- **Written/oral communication** — PLO5 (สื่อสาร)
- **Problem-solving** — PLO2 (แก้ปัญหาองค์กร/ชุมชน)
- **Team-building** — PLO5 (การทำงานร่วมกัน)
- **Lifelong learning** — PLO5 (เรียนรู้ตลอดชีวิต)
- **Professional ethics** — PLO4 (จริยาบรรณวิชาชีพ)

สำหรับบัณฑิตศึกษา PLOs ยังพิจารณา national/regional/global reference points ผ่านการออกแบบที่เน้น original research (PLO3) และ emerging technologies (PLO1) สอดคล้องกับ AUN-QA Addendum สำหรับหลักสูตรบัณฑิตศึกษา

### จุดแข็ง (Strengths)
- ครบถ้วนทั้ง Generic และ Subject-Specific ตามเกณฑ์ AUN-QA v4.0
- PLO5 ครอบคลุม Generic Outcomes หลายด้านในข้อเดียว เหมาะสมกับระดับบัณฑิตศึกษา

### จุดที่ต้องพัฒนา (Areas for Improvement)
- ควรแสดงตารางเปรียบเทียบ PLO กับ AUN-QA Generic Outcomes Framework อย่างชัดเจนในรายงาน

## Sub-criterion 1.4 — Stakeholder Requirements สะท้อนใน PLOs

### คำอธิบาย (Description)

หลักสูตรได้รวบรวมความต้องการจากผู้มีส่วนได้ส่วนเสีย (Stakeholders: SHs) โดยแบ่งออกเป็น 2 กลุ่มหลัก:

**SHs ภายนอก (External Stakeholders):**
| กลุ่ม | วิธีเก็บข้อมูล | PLO ที่สะท้อน |
|-------|---------------|--------------|
| ผู้ใช้บัณฑิต/สถานประกอบการด้าน Digital/AI ในภาคเหนือตอนล่าง | แบบสอบถาม + Focus Group | PLO1, PLO2, PLO3 |
| องค์กรปกครองส่วนท้องถิ่น (อปท.) จังหวัดอุตรดิตถ์และใกล้เคียง | การหารือ/มีส่วนร่วม | PLO2 |
| ศิษย์เก่า ป.ตรี วิศวกรรมคอมพิวเตอร์ | สนทนากลุ่ม | PLO1, PLO4 |

**SHs ภายใน (Internal Stakeholders):**
| กลุ่ม | วิธีเก็บข้อมูล | PLO ที่สะท้อน |
|-------|---------------|--------------|
| อาจารย์ผู้รับผิดชอบหลักสูตร/ผู้สอน | ประชุม กมศ. ทบทวน PLO/PEO | PLO1–PLO5 |
| นักศึกษา/ผู้สมัครเข้าศึกษา | ปฐมนิเทศ + เอกสารรับสมัคร | PLO1–PLO5 |

ความต้องการจาก SHs ถูกวิเคราะห์ จัดกลุ่ม และแปลงเป็น PLOs ผ่านกระบวนการ Needs Analysis โดยอ้างอิงเพิ่มเติมจาก AI Workforce Development Plan ของภาครัฐ เพื่อให้สะท้อนความต้องการของตลาดแรงงานในระดับภูมิภาคและระดับชาติ

> ⚠️ **หมายเหตุ (MARK):** ยังต้องการหลักฐานเชิงประจักษ์เพิ่มเติม ได้แก่ รายงานสรุปการสำรวจผู้ประกอบการ รายงานการสนทนากลุ่ม (Focus Group Minutes) และตารางแสดง Needs-to-PLO Mapping อย่างชัดเจน

### จุดแข็ง (Strengths)
- ระบุกลุ่ม SHs ครอบคลุมทั้งภายในและภายนอกมหาวิทยาลัย
- กระจายการรับฟังความต้องการหลายช่องทาง ครอบคลุมหลายมิติ

### จุดที่ต้องพัฒนา (Areas for Improvement)
- **ขาดหลักฐานเชิงประจักษ์** — ไม่มีรายงานสรุปเชิงปริมาณจากผู้ประกอบการ/ศิษย์เก่า
- ควรจัดทำตาราง "ความต้องการ SHs → PLO" อย่างชัดเจน (Needs-to-Outcomes Traceability Matrix)

## Sub-criterion 1.5 — PLOs บรรลุโดยผู้สำเร็จการศึกษา

### คำอธิบาย (Description)

หลักสูตรเป็น **หลักสูตรใหม่ พ.ศ. 2568** มีนักศึกษาเข้าศึกษา 4 คน (ภาคการศึกษาที่ 2/2568) จึงยังไม่มีบัณฑิตสำเร็จการศึกษา อย่างไรก็ตาม หลักสูตรได้วางระบบการประเมินการบรรลุ PLOs ใน 4 มิติดังนี้

| มิติ | รายละเอียด |
|------|-----------|
| **When** | ประเมินระหว่างเรียน (CLO assessment ทุกรายวิชา) + ก่อนสำเร็จการศึกษา (thesis/สารนิพนธ์ defense) |
| **Method** | CLO assessment ผ่านการสอบ/งาน + การประเมินวิทยานิพนธ์/สารนิพนธ์ + แบบสำรวจทักษะก่อนจบ |
| **Tool** | แบบประเมิน CLO รายวิชา, เกณฑ์ตรวจวิทยานิพนธ์ของคณะกรรมการ, แบบประเมิน PLO4/PLO5 |
| **Person** | อาจารย์ผู้สอน (CLO), คณะกรรมการสอบวิทยานิพนธ์/สารนิพนธ์ (PLO1/PLO3) |

แผนการติดตาม PLO Achievement:
1. รวบรวมผล CLO ทุกปลายภาคการศึกษา → จัดทำ PLO Dashboard
2. ใช้ผลการสอบป้องกันวิทยานิพนธ์/สารนิพนธ์ยืนยัน PLO1 และ PLO3
3. แบบประเมินตนเองสำหรับ PLO4 (จริยธรรม) และ PLO5 (ทักษะ Generic) ก่อนรับปริญญา
4. สำรวจบัณฑิตและผู้ใช้บัณฑิต 1 ปีหลังสำเร็จการศึกษา

> ⚠️ **ข้อจำกัดสำคัญ:** ยังไม่มีข้อมูลผลการบรรลุ PLOs จริง — คะแนน 2/7 ได้มาจากความสมบูรณ์ของการออกแบบระบบ ไม่ใช่ผลลัพธ์ที่วัดได้จริง

### จุดแข็ง (Strengths)
- ออกแบบระบบประเมิน PLO Achievement ครบทั้ง 4 มิติ (When/Method/Tool/Person)
- มีแผนชัดเจนสำหรับการติดตามผลระยะยาวผ่าน Alumni Tracking

### จุดที่ต้องพัฒนา (Areas for Improvement)
- **ยังไม่มีข้อมูลจริง** — ต้องรอจนกว่าจะมีบัณฑิตสำเร็จการศึกษา (ประมาณ พ.ศ. 2570)
- ควรเริ่มรายงาน CLO Attainment รายวิชาตั้งแต่ภาคแรกที่เปิดสอน เพื่อแสดง Trend ระหว่างทาง
